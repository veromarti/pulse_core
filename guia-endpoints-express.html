Endpoints con Express.js

Todo lo que necesitas para crear una API REST desde cero.

1

Instalación y configuración inicial

Instala Express y crea el archivo principal de tu servidor.

terminal
npm init -y
npm install express

index.js
const express = require('express');
const app = express();

// Permite recibir JSON en el body
app.use(express.json());

app.listen(3000, () => {
  console.log('Servidor corriendo en http://localhost:3000');
});
2

Los 4 métodos HTTP principales

GET

Obtener / leer datos

POST

Crear un nuevo recurso

PUT

Actualizar un recurso existente

DELETE

Eliminar un recurso

3

Estructura de un endpoint

Todo endpoint tiene esta forma básica:

app.método('/ruta', (req, res) => {
  // req → lo que llega (body, params, query)
  // res → lo que devuelves
  res.json({ mensaje: 'respuesta' });
});

Formas de enviar datos desde el cliente:

req.body
POST / PUT
Datos en el cuerpo
{ name: "Alice" }
req.params
Parte de la URL
/users/:id
req.params.id → "5"
req.query
Filtros en URL
/users?role=admin
req.query.role → "admin"
req.headers
Cabeceras HTTP
Authorization, etc.
req.headers.authorization
4

CRUD completo — ejemplo con usuarios

// GET — obtener todos los usuarios
app.get('/users', (req, res) => {
  res.json({ users: [] });
});

// GET — obtener un usuario por ID
app.get('/users/:id', (req, res) => {
  const { id } = req.params;
  res.json({ id });
});

// POST — crear un usuario
app.post('/users', (req, res) => {
  const { name, email } = req.body;
  res.status(201).json({ message: 'Usuario creado', name, email });
});

// PUT — actualizar un usuario
app.put('/users/:id', (req, res) => {
  const { id } = req.params;
  const data = req.body;
  res.json({ message: `Usuario ${id} actualizado`, data });
});

// DELETE — eliminar un usuario
app.delete('/users/:id', (req, res) => {
  const { id } = req.params;
  res.json({ message: `Usuario ${id} eliminado` });
});
5

Organizar con Router (buenas prácticas)

Cuando tienes muchos endpoints, sepáralos en archivos por recurso.

routes/users.js
const router = require('express').Router();

router.get('/', (req, res) => res.json({ users: [] }));
router.post('/', (req, res) => res.status(201).json({ message: 'Creado' }));
router.put('/:id', (req, res) => res.json({ message: 'Actualizado' }));
router.delete('/:id', (req, res) => res.json({ message: 'Eliminado' }));

module.exports = router;

index.js
const usersRouter = require('./routes/users');

// Todos los endpoints de users quedan bajo /users
app.use('/users', usersRouter);
💡 Con esto /users llama a router.get('/') y /users/5 llama a router.put('/:id') automáticamente.
6

Códigos de respuesta más usados

res.status(200).json({})  // OK — respuesta exitosa (default)
res.status(201).json({})  // Created — recurso creado exitosamente
res.status(400).json({})  // Bad Request — datos inválidos del cliente
res.status(401).json({})  // Unauthorized — no autenticado
res.status(403).json({})  // Forbidden — sin permisos
res.status(404).json({})  // Not Found — recurso no existe
res.status(500).json({})  // Internal Server Error — error en el servidor
